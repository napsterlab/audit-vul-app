§ Audit Methodology

This audit combined automated static analysis with manual code inspection using AST traversal and pattern matching. The following tools and techniques were employed:

Bandit
Python SAST scanner. Detects common security issues: B105 hardcoded passwords, B301 pickle, B324 MD5, B506 yaml, B602 subprocess shell, B608 SQLi.
AST Inspection
Python Abstract Syntax Tree traversal to detect f-string injection in SQL queries, render_template_string misuse, and JWT algorithm confusion.
Pattern Analysis
Regex and dataflow tracing to identify missing rate limiting, CORS wildcards, absent security headers, and sensitive data in HTTP responses.
Manual Inspection
Line-by-line code review focused on authorization logic, input validation, deserialization entry points, and business logic flaws like negative transfers.
OWASP Reference
All findings mapped to OWASP Top 10 (2021) and CWE identifiers. CVSS v3.1 base scores calculated for each vulnerability independently.
Threat Modeling
Attack vectors constructed and verified for each finding, ensuring exploitability is confirmed before severity classification is assigned.
§ Secure Coding Best Practices
✓ Always use parameterized queries
Never concatenate user data into SQL strings. Use ? placeholders for all inputs.
✓ Never deserialize untrusted data
Avoid pickle, YAML unsafe-load, and Java's ObjectInputStream on any user-controlled input.
✓ Use secrets from environment
All secret keys, passwords, and tokens must be loaded at runtime from environment variables or a secrets manager.
✓ Validate & sanitize all input
Use strict schemas with type enforcement. Reject requests that contain unexpected fields or values outside allowed ranges.
✓ Principle of least privilege
Each user and service account should only access resources explicitly authorized for their role. Check on every request.
✓ Deploy with security headers
CSP, HSTS, X-Frame-Options, and X-Content-Type-Options are non-optional for any web application serving browsers.
