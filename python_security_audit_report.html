Immediate Actions (0–48 hrs)
🔴Disable debug mode — set DEBUG=False, load secrets from env vars, never hardcode credentials.
🔴Patch SQL injection — replace all f-string queries with parameterized cursor.execute(query, (params,)).
🔴Remove pickle/yaml.load — use json for session data, yaml.safe_load() for config files.
🔴Fix command injection — pass args as list, never use shell=True with user input.
🔴Fix JWT algorithm confusion — restrict to algorithms=["HS256"] only, reject none.
Short-term (1–2 weeks)
🟠Replace MD5 with bcrypt or argon2 for password hashing with salting.
🟠Add authorization checks — verify request.user['id'] == user_id before returning any account data.
🟠Implement rate limiting — use flask-limiter on /login and /transfer routes.
🟠Fix path traversal — use os.path.abspath() + commonpath() to validate file paths stay in allowed directory.
🟠Sanitize template input — use render_template() with static files instead of render_template_string with user data.
🟠Replace xml.etree with defusedxml — prevents XXE and billion laughs attacks.
Long-term (2–4 weeks)
🟡Add security headers — CSP, X-Frame-Options, HSTS, X-Content-Type-Options via flask-talisman.
🟡Fix CORS policy — restrict origin to known domains, never use * for authenticated APIs.
🟡Redact logs — never log passwords, tokens, or PII; use structured logging with field masking.
🟡Input validation — add Pydantic schemas or marshmallow for all request bodies; enforce field allowlists.
🟡Integrate CI scanning — add Bandit + Semgrep to GitHub Actions, block PRs on high-severity findings.
🟡Secret management — move all secrets to HashiCorp Vault, AWS Secrets Manager, or at minimum .env + python-dotenv.
